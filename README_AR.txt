دليل آية للحمل V4.0.0 — نسخة نبض اليومية + Supabase
====================================================

هذه النسخة هي مشروع كامل جاهز للرفع إلى GitHub ثم الربط بمشروع Vercel الحالي guideayah.

طريقة الرفع إلى GitHub:
1) فك ضغط الملف النهائي.
2) افتح المجلد الداخلي guideayah-v4.
3) ارفع محتويات المجلد إلى جذر مستودع GitHub، وليس ملف ZIP نفسه.
4) يجب أن تظهر ملفات مثل index.html وstyles.css ومجلدات api وjs وassets مباشرة في جذر المستودع.

ربط Vercel:
1) افتح Vercel → مشروع guideayah → Settings → Git.
2) اربط مستودع GitHub الذي رفعت إليه هذه الملفات.
3) فرع الإنتاج: main.
4) Framework Preset: Other إذا طلب منك.
5) لا تحتاج Build Command ولا Output Directory.

متغيرات OpenAI في Vercel:
- OPENAI_API_KEY = مفتاح OpenAI الخاص بك.
- OPENAI_MODEL = gpt-5.6 (اختياري حسب النموذج المتاح لحسابك).

متغيرات Supabase في Vercel:
- SUPABASE_URL = رابط مشروع Supabase.
- SUPABASE_ANON_KEY = المفتاح العام Publishable/Anon فقط.

بعد إضافة أو تعديل أي متغير بيئة، نفذ Redeploy.

لتفعيل Google والمزامنة:
- اتبع ملف SUPABASE_SETUP_AR.md.
- نفذ ملف supabase/schema.sql داخل SQL Editor في Supabase.
- فعّل Google Provider وأضف روابط guideayah.vercel.app في URL Configuration.

أهم مزايا V4:
- صفحة رئيسية جديدة بألوان وردية/كريمية وصورة الحامل والشعار الجديد.
- تحية ورسالة يوم متغيرة باسم نبض.
- 6 حالات سريعة: مرتاحة، غثيان، تعبانة، ما لي خلق، جوعانة، أبغى اهتمام.
- اختيار يومي يتغير حسب الحالة مع زر «غيّري لي اليوم».
- مفاجأة نبض ورسائل أحمد الخاصة.
- رسالة يومية من البيبي مرتبطة بمرحلة الحمل.
- مذكرات نبض والذكريات والصور المحلية.
- إعدادات حمل موسعة: اسم البيبي، عدد الأجنة، الطبيبة والعيادة.
- تخصيص شخصية التطبيق: طول الرسائل، الأسلوب، مستوى وأوقات التذكير، الأطعمة المحببة والمنفرة، الروائح والأعراض المتكررة.
- تخصيص الوجبات محليًا وفق الغثيان والتفضيلات والنفور من الطعام.
- تطوير مساعد آية وإرسال سياق التخصيص الضروري فقط.
- أوامر محلية داخل المساعد مثل «شربت كوبين» و«أخذت حمض الفوليك» لتحديث التطبيق مباشرة.
- حفظ محلي مستمر دون تسجيل.
- تسجيل Google اختياري عبر Supabase Auth.
- مزامنة إعدادات الحمل والمتابعة اليومية والمذكرات والمواعيد والتفضيلات بين الأجهزة.
- سياسات RLS لعزل بيانات كل مستخدمة.
- PWA يعمل دون اتصال للميزات المحلية.

ملاحظات صريحة:
- صور الذكريات تبقى في IndexedDB على الجهاز في V4 ولا تتزامن مع Supabase بعد.
- تسجيل Google الحقيقي يحتاج إعداد Google OAuth وSupabase ومتغيرات Vercel؛ لا يمكن اختباره دون بيانات المشروع الفعلية.
- التطبيق يظل يعمل بالحفظ المحلي حتى لو لم تفعّل Supabase.
